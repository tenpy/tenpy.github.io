Engine¶

  • full name: tenpy.algorithms.dmrg.Engine

  • parent module: tenpy.algorithms.dmrg

class tenpy.algorithms.dmrg.Engine[source]¶

Bases: tenpy.linalg.sparse.NpcLinearOperator

Prototype for an DMRG ‘Engine’.

This class is deprecated with the arrival of the new Sweep-based classes.

The old docstring:

This class is the working horse of DMRG. It implements the sweep() and large parts of the (two-site) optimization. During the diagonalization (i.e. after calling prepare_diag()), the class represents the effective two-site Hamiltonian, which looks like this:

|        .---            ----.
|        |     |      |      |
|        LP----W[i0]--W[i1]--RP
|        |     |      |      |
|        .---            ----.

LP and RP are left and right parts of the MPOEnvironment, W[i0] and W[i1] are the MPO matrices of the Hamiltonian at the two sites i0, i1=i0+1. How this network is then actually contracted in detail is left to derived classes.

Attributes
verboseint

Level of verbosity (i.e. how much status information to print); higher=more output.

psiMPS

The MPS to be optimized. After run(), this should be (close to) the ground state.

sweepsint

The number of performed sweeps (with optimize=True, i.e. not counting the environment updates).

envMPOEnvironment

Environment for contraction <psi|H|psi>.

ortho_to_envslist of MPSEnvironment

Environments of the form <psi|orhto> for states ortho of the orthogonal_to parameter; needed to find excited states.

mixerMixer | None

If None, no mixer is used (anymore), otherwise the mixer instance.

DMRG_paramsdict

Parameters used for the DMRG. See run() for more details.

lanczos_paramsdict

Parameters for lanczos().

trunc_paramsdict

Parameters for truncate().

finitebool

Wheter we perform DMRG for an MPS with finite/segment (True) or infinite boundary conditions; same as psi.finite.

shelvebool

True when the DMRG stopped due to the time limit set by max_hours, otherwise False.

chi_listdict | None

See DMRG_params chi_list. None (default) disables this feature.

update_statsdict

matvec(vec)¶

Calculate the action of the operator on a vector vec.

Note that we don’t require vec to be one-dimensional. However, for square operators we require that the result of matvec has the same legs (in the same order) as vec such that they can be added. Note that this excludes a non-trivial qtotal for square operators.
